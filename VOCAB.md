# 2026-05-22
- Frontend: program that runs after the browser loads in all the assests. Handles UI visualizations and user interactions
- Backend: program running on the server that handles requests and communicates with the database
- Database: system that stores data and allows reads/writes from the program
- Worker: a separate program from the backend that handles slower jobs. Allows backend to continue running without long blockages
- Queue: data structure that holds messages in order to be processed. Producers write them and consumers read and process them
- Request/Response: basic pattern of HTTP communication.
    - Request is sent by the client specifying a method (GET, POST, PUT, DELETE)
    - Response is the server replying back
    * Servers only ever reply when prompted by client
- CORS (Cross Origin Resource Sharing): mechanism that allows legit cross-origin requests. Website tells browser that the page from that origin is allowed to make requests.
- Same-Origin Policy: JavaScript running on one origin cannot request another
    - Prevents malicious sites from using your cookie in your browser to request other origins
- Client / Server: roles in a request, not specific programs. The client initiates the request; the server responds. The same program can be a client in one interaction and a server in another.

# 2026-05-23
- .gitignore: a file listing specific files Git should never track/contain. Including those files may cause security issues or future merge conflicts.
- .lock file: autogenerated file listing exact versions of dependencies that coded against. Ensures reproducibility across different machines

# 2026-05-24
- Untracked file: a file in repo that Git isn't watching yet. Starts being tracked once you git add it.
- Tracked: a file Git is monitoring for changes
- Branch: a named line of dev in Git. By default you start on main, then you create new branches which can later merge.
